s2n-tls is an open-source TLS/SSL library from AWS designed with security as a priority. It implements TLS 1.0 through TLS 1.3, supports post-quantum cryptography, and is used in production across AWS services.

> *s2n-tls is short for "signal to noise" — a nod to the act of encryption, disguising meaningful signals as seemingly random noise.*

## Why s2n-tls?

<CardGroup cols={2}>
  <Card title="Minimal attack surface" icon="minimize">
    Avoids rarely-used options and features with a history of protocol-level vulnerabilities
  </Card>
  <Card title="Post-quantum ready" icon="atom">
    ML-KEM key exchange and ML-DSA authentication for quantum-resistant TLS 1.3
  </Card>
  <Card title="Multiple crypto backends" icon="layer-group">

  </Card>
  <Card title="FIPS support" icon="certificate">

## Key features

- **TLS 1.0 – 1.3** with sensible defaults and versioned security policies
- **Session resumption** and **pre-shared keys** to reduce handshake cost
- **Certificate authentication** including mutual TLS, OCSP stapling, and CRL validation
- **Timing blinding** to defeat side-channel attacks
- **Memory protection** — plaintext erased on read, mlock prevents swap
- **No locks or mutexes** — safe for multi-threaded applications

## Get started

<Steps>
  <Step title="Build s2n-tls">
    Install dependencies and build the library with CMake. See the [build guide](/building).
  </Step>
  <Step title="Initialize the library">
    Call `s2n_init()` once at startup before using any s2n-tls functions.
  </Step>
  <Step title="Create a config and connection">
    Use `s2n_config_new()` to set certificates and security policy, then `s2n_connection_new()` to create a connection.
  </Step>
  <Step title="Negotiate and send data">
    Call `s2n_negotiate()` to perform the TLS handshake, then use `s2n_send()` and `s2n_recv()` to exchange application data.
  </Step>
</Steps>
