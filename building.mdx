---
title: "Build & install"
description: "Build s2n-tls from source on Ubuntu, macOS, or Amazon Linux 2 and integrate it into your CMake project."
---

s2n-tls uses CMake as its primary build system. The instructions below cover installing dependencies, building the library, running tests, and installing the artifacts.

<Note>
  s2n-tls does not support building on Windows. See [issue #497](https://github.com/aws/s2n-tls/issues/497) for tracking information.
</Note>

## Install dependencies and build

<Tabs>
  <Tab title="Ubuntu">
    ```bash
    # Install build dependencies
    sudo apt update
    sudo apt install cmake

    # Install a libcrypto (OpenSSL is used here; see below for AWS-LC)
    sudo apt install libssl-dev

    # Configure, build, test, and install
    cmake . -Bbuild \
        -DCMAKE_BUILD_TYPE=Release \
        -DCMAKE_INSTALL_PREFIX=./s2n-tls-install
    cmake --build build -j $(nproc)
    CTEST_PARALLEL_LEVEL=$(nproc) ctest --test-dir build
    cmake --install build
    ```
  </Tab>
  <Tab title="macOS">
    ```bash
    # Install build dependencies
    brew install cmake

    # Install a libcrypto
    brew install openssl@3

    # Configure, build, test, and install
    cmake . -Bbuild \
        -DCMAKE_BUILD_TYPE=Release \
        -DCMAKE_PREFIX_PATH=$(dirname $(dirname $(brew list openssl@3 | grep libcrypto.dylib))) \
        -DCMAKE_INSTALL_PREFIX=./s2n-tls-install
    cmake --build build -j $(sysctl -n hw.ncpu)
    CTEST_PARALLEL_LEVEL=$(sysctl -n hw.ncpu) ctest --test-dir build
    cmake --install build
    ```
  </Tab>
  <Tab title="Amazon Linux 2">
    ```bash
    # Install build dependencies
    sudo yum groupinstall "Development Tools"
    sudo yum install cmake3

    # Install a libcrypto
    sudo yum install openssl-devel

    # Configure, build, test, and install
    cmake3 . -Bbuild \
        -DCMAKE_BUILD_TYPE=Release \
        -DCMAKE_INSTALL_PREFIX=./s2n-tls-install \
        -DCMAKE_EXE_LINKER_FLAGS="-lcrypto -lz"
    cmake3 --build build -j $(nproc)
    cd build && CTEST_PARALLEL_LEVEL=$(nproc) ctest3 && cd ..
    cmake3 --install build
    ```
  </Tab>
</Tabs>

After installation the library headers and compiled artifacts are in `./s2n-tls-install`. The `s2nc` and `s2nd` test utilities are built in `build/bin` but are not installed by default. To install them, add `-DS2N_INSTALL_S2NC_S2ND=1` to the `cmake` configure command.

## CMake options

Each option is passed as `-D<option>=<value>` on the CMake command line.

| Option | Description |
|---|---|
| [`CMAKE_BUILD_TYPE`](https://cmake.org/cmake/help/latest/variable/CMAKE_BUILD_TYPE.html) | `Release` for optimized production builds; `Debug` for debug symbols and GDB support. |
| [`CMAKE_INSTALL_PREFIX`](https://cmake.org/cmake/help/latest/variable/CMAKE_INSTALL_PREFIX.html) | Directory where library artifacts and headers are placed on `cmake --install`. |
| [`CMAKE_PREFIX_PATH`](https://cmake.org/cmake/help/latest/variable/CMAKE_PREFIX_PATH.html) | Colon-separated list of install prefixes CMake searches for dependencies. Use this to point s2n-tls at a specific libcrypto installation. |
| [`BUILD_SHARED_LIBS`](https://cmake.org/cmake/help/latest/variable/BUILD_SHARED_LIBS.html) | `OFF` (default) builds a static library. Set to `ON` to build a shared library instead. |

To view all available s2n-tls CMake options with descriptions, run:

```bash
cmake . -LH
```

## Building with a specific libcrypto

s2n-tls requires one of the following libcrypto implementations to be linked at build time. By default CMake searches the system for a compatible library. Override this by setting `CMAKE_PREFIX_PATH` to the desired libcrypto install prefix.

### AWS-LC (recommended)

[AWS-LC](https://github.com/aws/aws-lc) is the recommended libcrypto for s2n-tls. It provides better performance, sandboxing support, post-quantum key exchange, and FIPS 140-2/3 validation.

```bash
# 1. Build and install AWS-LC
cmake /path/to/aws-lc -Bbuild-awslc \
    -DCMAKE_BUILD_TYPE=Release \
    -DCMAKE_INSTALL_PREFIX=./awslc-install
cmake --build build-awslc -j $(nproc)
cmake --install build-awslc

# 2. Build s2n-tls pointing at the AWS-LC install
cmake . -Bbuild \
    -DCMAKE_BUILD_TYPE=Release \
    -DCMAKE_PREFIX_PATH=./awslc-install \
    -DCMAKE_INSTALL_PREFIX=./s2n-tls-install
cmake --build build -j $(nproc)
cmake --install build
```

See the [AWS-LC build documentation](https://github.com/aws/aws-lc/blob/main/BUILDING.md) for full AWS-LC build instructions.

### OpenSSL

OpenSSL versions 1.0.2 through 3.0 are supported. Note the following restrictions:

- ChaChaPoly and RSA-PSS require OpenSSL 1.1.1 or later.
- RC4 is not supported with OpenSSL 3.0 or later.
- FIPS mode is supported with FIPS-validated OpenSSL 3.0, but AWS-LC is recommended for FIPS requirements.

```bash
cmake . -Bbuild \
    -DCMAKE_BUILD_TYPE=Release \
    -DCMAKE_PREFIX_PATH=/path/to/openssl-install \
    -DCMAKE_INSTALL_PREFIX=./s2n-tls-install
cmake --build build -j $(nproc)
cmake --install build
```

### BoringSSL

[BoringSSL](https://boringssl.googlesource.com/boringssl) is supported with two caveats: OCSP features are not available, and FIPS mode is not supported.

```bash
cmake . -Bbuild \
    -DCMAKE_BUILD_TYPE=Release \
    -DCMAKE_PREFIX_PATH=/path/to/boringssl-install \
    -DCMAKE_INSTALL_PREFIX=./s2n-tls-install
cmake --build build -j $(nproc)
cmake --install build
```

### LibreSSL

[LibreSSL](https://www.libressl.org/) is supported as a drop-in libcrypto. Point `CMAKE_PREFIX_PATH` at the LibreSSL install directory.

## Consuming s2n-tls via CMake `find_package`

s2n-tls ships CMake finder scripts. After installing the library, integrate it into your own CMake project with two lines:

```cmake
find_package(s2n REQUIRED)

target_link_libraries(your_target PRIVATE AWS::s2n)
```

When configuring your project, tell CMake where s2n-tls is installed using one of two approaches:

<CodeGroup>

```bash local install
# Pass the s2n-tls install prefix
cmake /path/to/your/project -Bbuild \
    -DCMAKE_PREFIX_PATH=/path/to/s2n-tls-install
```

```bash system install
# If s2n-tls is installed system-wide, no extra flags are needed
cmake /path/to/your/project -Bbuild
```

</CodeGroup>

## mlock limits

s2n-tls calls `mlock()` internally to prevent plaintext memory from being swapped to disk. On some systems the default limit on locked memory is low enough to cause build tests to fail.

Check your current limit with:

```bash
ulimit -l
```

To raise the limit, consult your OS documentation (e.g., `/etc/security/limits.conf` on Linux).

### Disabling mlock

If you cannot raise the limit, you can disable s2n-tls's `mlock()` behavior by setting an environment variable before running your application or the test suite:

```bash
S2N_DONT_MLOCK=1 ./your_application
```

<Warning>
  Disabling `mlock()` means plaintext TLS data may be swapped to disk. Only use `S2N_DONT_MLOCK=1` in development environments where this risk is acceptable.
</Warning>

## Other build methods

<AccordionGroup>
  <Accordion title="Ninja build system">
    [Ninja](https://ninja-build.org/) can improve build performance when used as the CMake generator.

    ```bash
    # Ubuntu
    sudo apt install ninja-build

    cmake . -Bbuild -GNinja \
        -DCMAKE_BUILD_TYPE=Release \
        -DCMAKE_INSTALL_PREFIX=./s2n-tls-install
    ninja -C build -j $(nproc)
    CTEST_PARALLEL_LEVEL=$(nproc) ninja -C build test
    ninja -C build install
    ```
  </Accordion>
  <Accordion title="Traditional Makefile">
    Building with a traditional Makefile is supported on some platforms. Set the `LIBCRYPTO_ROOT` environment variable to the libcrypto install directory.

    ```bash
    LIBCRYPTO_ROOT=/usr/local/ssl make
    ```

    CMake is preferred because it includes more features and broader platform support.
  </Accordion>
  <Accordion title="Cross-compiling for 32-bit">
    An example toolchain file for 32-bit cross-compilation is provided at `cmake/toolchains/32-bit.toolchain`. You will need a 32-bit libcrypto.

    ```bash
    # Ubuntu: add the i386 architecture and install the 32-bit libcrypto
    sudo dpkg --add-architecture i386
    sudo apt update
    sudo apt install cmake clang gcc-multilib libssl-dev:i386

    # Build with the toolchain
    rm -rf build
    cmake . -Bbuild -DCMAKE_TOOLCHAIN_FILE=cmake/toolchains/32-bit.toolchain
    cmake --build ./build -j $(nproc)
    CTEST_PARALLEL_LEVEL=$(nproc) ctest --test-dir build
    ```
  </Accordion>
  <Accordion title="Nix development environment">
    s2n-tls supports Nix for development, providing an environment that closely matches CI.

    ```bash
    nix develop
    configure; build
    ```

    See the [s2n-tls Nix documentation](https://github.com/aws/s2n-tls/blob/main/nix/README.md) for setup instructions.
  </Accordion>
</AccordionGroup>
