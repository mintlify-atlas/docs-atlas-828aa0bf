## Loading certificates

```c
struct s2n_cert_chain_and_key *s2n_cert_chain_and_key_new(void);

int s2n_cert_chain_and_key_load_pem_bytes(
    struct s2n_cert_chain_and_key *chain_and_key,
    uint8_t *chain_pem, uint32_t chain_pem_len,
    uint8_t *private_key_pem, uint32_t private_key_pem_len);

int s2n_cert_chain_and_key_load_pem(
    struct s2n_cert_chain_and_key *chain_and_key,
    const char *chain_pem, const char *private_key_pem);

int s2n_cert_chain_and_key_free(struct s2n_cert_chain_and_key *cert_and_key);
```

```c
struct s2n_cert_chain_and_key *cert = s2n_cert_chain_and_key_new();
s2n_cert_chain_and_key_load_pem_bytes(cert,
    (uint8_t *)cert_pem, cert_pem_len,
    (uint8_t *)key_pem, key_pem_len);
s2n_config_add_cert_chain_and_key_to_store(config, cert);
/* ... use config ... */
s2n_cert_chain_and_key_free(cert);
```

## Adding to a config

```c
int s2n_config_add_cert_chain_and_key_to_store(struct s2n_config *config,
    struct s2n_cert_chain_and_key *cert_key_pair);

int s2n_config_set_cert_chain_and_key_defaults(struct s2n_config *config,
    struct s2n_cert_chain_and_key **cert_key_pairs,
    uint32_t num_cert_key_pairs);
```

Servers may load multiple certificates (RSA, RSA-PSS, ECDSA). s2n-tls selects the appropriate certificate based on the client's SNI extension.

## SNI

```c
int s2n_set_server_name(struct s2n_connection *conn, const char *server_name);
```

Call on client connections before `s2n_negotiate()` to set the SNI extension value. The server uses this to select the matching certificate.

## Host verification callback

```c
typedef uint8_t (*s2n_verify_host_fn)(const char *host_name, size_t host_name_len,
    void *data);

int s2n_config_set_verify_host_callback(struct s2n_config *config,
    s2n_verify_host_fn, void *data);

int s2n_connection_set_verify_host_callback(struct s2n_connection *conn,
    s2n_verify_host_fn, void *data);
```

Called for each name in the peer's certificate. Return `1` to accept, `0` to reject.
