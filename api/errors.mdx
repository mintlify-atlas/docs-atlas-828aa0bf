## Return values

s2n-tls functions signal failure as follows:
- Functions returning `int`: return `S2N_FAILURE` (`-1`) on error, `S2N_SUCCESS` (`0`) on success
- Functions returning pointers: return `NULL` on error

When a function fails, `s2n_errno` is set to the error code.

## s2n_errno

```c
S2N_API extern __thread int s2n_errno;
S2N_API extern int *s2n_errno_location(void);
```

`s2n_errno` is thread-local. Use `s2n_errno_location()` when accessing from languages other than C to ensure the correct TLS mechanism is used.

<Note>
Clear `s2n_errno` after handling an error: `s2n_errno = S2N_ERR_T_OK`
</Note>

## Error description functions

```c
const char *s2n_strerror(int error, const char *lang);
const char *s2n_strerror_name(int error);
const char *s2n_strerror_debug(int error, const char *lang);
const char *s2n_strerror_source(int error);
```

- `s2n_strerror()` — human-readable error message (pass `"EN"` for English)
- `s2n_strerror_name()` — symbolic error name (e.g. `"S2N_ERR_CERT_EXPIRED"`)
- `s2n_strerror_debug()` — debug string including filename and line number
- `s2n_strerror_source()` — source file where the error originated

## Error types

```c
int s2n_error_get_type(int error);
```

Returns one of these `s2n_error_type` values:

| Type | Value | Meaning |
|------|-------|---------|
| `S2N_ERR_T_OK` | 0 | No error |
| `S2N_ERR_T_IO` | 1 | Underlying I/O error — check `errno` |
| `S2N_ERR_T_CLOSED` | 2 | Connection closed |
| `S2N_ERR_T_BLOCKED` | 3 | Operation would block — retry |
| `S2N_ERR_T_ALERT` | 4 | TLS alert received from peer |
| `S2N_ERR_T_PROTO` | 5 | TLS protocol error |
| `S2N_ERR_T_INTERNAL` | 6 | Internal s2n-tls error |
| `S2N_ERR_T_USAGE` | 7 | Incorrect API usage |

```c
s2n_errno = S2N_ERR_T_OK;
if (s2n_negotiate(conn, &blocked) < 0) {
    switch (s2n_error_get_type(s2n_errno)) {
        case S2N_ERR_T_BLOCKED:
            return RETRY;
        case S2N_ERR_T_CLOSED:
            return SUCCESS;
        case S2N_ERR_T_IO:
            handle_io_err(errno);
            return FAILURE;
        case S2N_ERR_T_PROTO:
            handle_proto_err();
            return FAILURE;
        case S2N_ERR_T_ALERT:
            log_alert(s2n_connection_get_alert(conn));
            return FAILURE;
        default:
            log_other_error();
            return FAILURE;
    }
}
```

## Alerts

```c
int s2n_connection_get_alert(struct s2n_connection *conn);
```

When `s2n_error_get_type()` returns `S2N_ERR_T_ALERT`, call this to get the specific TLS alert code received from the peer.

## Blinding

Blinding delays responses after peer-triggered errors to prevent timing side-channel attacks.

```c
typedef enum {
    S2N_BUILT_IN_BLINDING,
    S2N_SELF_SERVICE_BLINDING,
} s2n_blinding;

int s2n_connection_set_blinding(struct s2n_connection *conn, s2n_blinding blinding);
uint64_t s2n_connection_get_delay(struct s2n_connection *conn);
int s2n_config_set_max_blinding_delay(struct s2n_config *config, uint32_t seconds);
```

**Built-in blinding** (default): s2n-tls sleeps 10–30 seconds on peer errors, blocking the calling thread.

**Self-service blinding**: The application manages the delay. After a peer-triggered error, call `s2n_connection_get_delay()` and pause the connection for that many nanoseconds before calling `s2n_shutdown()` or `close()`.

<Warning>
Failing to implement self-service blinding correctly (e.g. not waiting the full delay) makes your application vulnerable to timing side-channel attacks.
</Warning>

## Stack traces

Enable stack traces to debug errors:

```bash
# Enable via environment variable
S2N_PRINT_STACKTRACE=1 ./your-application
```

```c
// Or enable in code
s2n_stack_traces_enabled_set(true);
s2n_print_stacktrace(stderr);
```

<Note>
Stack traces significantly slow down unit tests and time-sensitive code paths.
</Note>
