## Crate descriptions

### s2n-tls

The primary crate for most users. Provides safe, idiomatic Rust wrappers around the s2n-tls C library:
- `Config` — wraps `s2n_config`
- `Connection` — wraps `s2n_connection`
- Certificate loading helpers
- Security policy configuration

### s2n-tls-tokio

Async TLS for Tokio applications. Provides `TlsAcceptor` and `TlsConnector` that wrap `tokio::net::TcpStream` into async TLS streams compatible with `tokio::io::AsyncRead` / `AsyncWrite`.

### s2n-tls-sys

Auto-generated FFI bindings directly to the C API. Use this only when you need access to functions not yet exposed by the high-level `s2n-tls` crate.

## Workspace structure

```
bindings/rust/
├── standard/        # s2n-tls and s2n-tls-sys crates
└── extended/        # s2n-tls-tokio and other extended crates
```
